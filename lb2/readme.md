# Лабораторная работа 2. Обратный алгоритм RSK и построение класса эквивалентности по Кнутту

## Условие задачи

### Часть 1. Обратный алгоритм RSK

**Вход:** 2 файла (содержащие таблицы P и Q).

**Формат файлов:** в каждой строке через пробел перечислены координаты клетки (нумерация от 0) и записанное в клетке число. Т. е. файл состоит из трёх столбцов: x, y, содержимое клетки.

**Результат**: Текстовый файл, содержащий исходную перестановку. Перестановка записана в одной строке через пробел. 

**Пример работы**: 

Вход:

```
P.txt:
0 0 1
0 1 2
0 2 4
1 0 3

Q.txt:
0 0 1
0 1 3
0 2 4 
1 0 2
```

Вывод:

```3 1 2 4``` 


### Часть 2. Построение классов эквивалентности по Кнуту

**Вход:** Текстовый файл `input_permutation.txt`, содержащий одну перестановку (целые числа через пробел в одной строке).

**Вывод:** Текстовый файл `knuth.txt` — все перестановки, эквивалентные по Кнуту исходной (т.е. имеющие ту же P-таблицу по RSK);

**Пример работы:**

Вход:

```
permutation.txt:
5 3 2 1 4
```

Вывод:
```
knuth.txt:
3 5 4 2 1
5 3 2 1 4
5 3 2 4 1
5 3 4 2 1
```

## Запуск

1. Клонировать репозиторий: 

```git clone git@github.com:azazzze1/CombMath_Shushkov.git```

2. Зайти в папку lb2:

```cd CombMath_Shushkov/lb2```

3. Создать необходимые файлы:
- ```pTest.txt```, ```qTest.txt``` — для обратного RSK.
- ```permutation.txt``` — для класса эквивалентности по Кнутту. 

4. Запустить makefile и программу:

```make && ./lab```
